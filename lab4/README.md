# Лабораторная работа 4: Защита программ от несанкционированного копирования

## Цель работы
Исследование способов защиты программ от несанкционированного копирования путём создания лицензионной системы, привязанной к аппаратной и программной конфигурации компьютера.

## Что реализовано

### 1. Сбор информации о системе
Программа собирает следующую информацию о компьютере с помощью встроенных python модулей:

- **ОС**: операционная система (Windows/Linux/macOS)
- **Версия**: версия ОС
- **Процессор**: название и модель CPU
- **ОЗУ**: объём оперативной памяти
- **Сетевой интерфейс**: MAC адрес
- **Имя компьютера**: hostname
- **Диски**: установленные диски (для Windows)

### 2. Создание "отпечатка" системы (System Fingerprint)
На основе собранной информации создаётся уникальный отпечаток системы:
```
MAC_ADDRESS|CPU|HOSTNAME|SYSTEM|ARCHITECTURE
```

### 3. Хеширование
Отпечаток хешируется функцией MD5 (реализированной на лабораторной работе 3) для получения лицензионного ключа.

### 4. Проверка лицензии
При запуске программа:
1. Генерирует текущий хеш системы
2. Сравнивает его с сохранённым хешем в файле `license.key`
3. Если хеши совпадают → программа работает
4. Если хеши не совпадают → выдаёт ошибку о нелегальном использовании

## Особенности реализации

### ✓ Привязка к аппаратному обеспечению
- MAC адрес сетевой карты
- Характеристики процессора
- Объём памяти
- Конфигурация дисков

### ✓ Привязка к программному обеспечению
- Версия операционной системы
- Архитектура системы (32-bit/64-bit)

### ✓ Защита от копирования
Если программу скопировать на другой компьютер, то:
- Система соберёт другую информацию
- Хеш будет отличаться
- Программа откажет в работе с сообщением об ошибке

### ✓ Без сторонних библиотек
Использованы только встроенные модули Python:
- `platform` — информация об ОС
- `socket` — работа с сетью
- `uuid` — получение MAC адреса
- `os` — работа с файловой системой
- `subprocess` — вызов системных команд (для получения дополнительной информации)
- `pathlib` — работа с путями

## Запуск

### 1. Первый запуск (генерирование лицензии)
```powershell
# Активировать виртуальное окружение
& .\.venv\Scripts\Activate.ps1

# Запустить программу
python main.py

На первом запуске программа автоматически:
- Собирает информацию о текущей системе
- Генерирует хеш (лицензионный ключ)
- Сохраняет его в файле `license.key`

При последующих запусках на той же системе:

**Если лицензия действительна (успех):**
```

======================================================================

**Если программу скопировали на другой компьютер (ошибка):**
```

Программа отказывает в работе.
```

## Как работает защита

### Сценарий 1: Программа работает нормально (на лицензированном ПК)
```
ПК 1 (лицензированный):
1. Первый запуск → генерирование license.key с хешем системы ПК1
2. Последующие запуски → хеш текущей системы == хеш в license.key
3. ✓ Программа запускается
```

### Сценарий 2: Попытка копирования программы
```
С ПК 1 → копируем папку с программой на ПК 2

ПК 2 (другой компьютер):
1. Запуск программы
2. Программа считывает license.key (с хешем от ПК1)
3. Программа генерирует текущий хеш системы ПК2
4. хеш ПК1 != хеш ПК2
5. ✗ ОШИБКА: "Программа не лицензирована на этом компьютере"
6. Программа отказывает в работе
```

## Структура функций

### `get_system_info()` → dict
Собирает информацию о системе

### `create_system_fingerprint(info)` → str
Создаёт текстовый отпечаток из информации системы

### `get_license_hash()` → str
Возвращает MD5 хеш текущего отпечатка

### класс `LicenseManager`
- `generate_license()` — генерирует лицензию для текущей системы
- `validate_license()` — проверяет валидность лицензии
- `get_system_info_display()` — выводит информацию о системе

### `md5(message: bytes)` → str
Функция хеширования MD5 (из лабораторной работы 3)

## Примечания

### Безопасность
- Программа не использует простые последовательности символов, а полноценное хеширование
- Невозможно подделать лицензию без знания точной конфигурации системы
- Смена даже одного компонента (процессор, материнская плата) приводит к неработоспособности

### Масштабируемость
Этот подход можно расширить:
- Добавить онлайн-проверку лицензии на сервере
- Реализовать срок действия лицензии
- Добавить разные уровни доступа
- Проверять дополнительные параметры (видеокарта, жёсткий диск и т.д.)

### Ограничения
- Легитимное обновление некоторых компонентов потребует перегенерировки лицензии
- Требует доступ к системной информации
- В некоторых окружениях вывод системной информации может быть ограничен
